## Remote Browser Configuration

Browserable supports various browser automation providers and configurations.

### Current Configuration

The default setup includes:
- Local browser automation (using Steel)
- Integration with popular remote browser providers (Hyperbrowser, Steel and Browserbase)
- Concurrent browser management
- Session handling

We are working on detailed documentation covering:
- Browser provider selection
- Scaling strategies
- Performance optimization
- Security configurations
